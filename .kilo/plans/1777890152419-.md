# Glassmorphic Active Card Styling Plan

## Goal

Implement consistent glassmorphic active card styling across all onboarding and welcome screens, using the LanguageSelectionScreen's active card style as the master reference.

## Current State Analysis

The codebase already has:

1. `src/lib/glass.tsx` - Provides foundational glassmorphism layers (wash, gradients, borders, shadows)
2. `LanguageSelectionScreen.tsx` - Already implements good active/inactive state styling with:
   - Soft translucent background (`rgba(255,255,255,0.18)` inactive, `rgba(255,255,255,0.38)` active)
   - Semi-transparent borders (`rgba(255,255,255,0.15)` inactive, `rgba(255,255,255,0.8)` active)
   - Proper scaling and visual emphasis changes
   - However, it uses direct styling without reusable components or animated transitions

## Updated Implementation Plan

### 1. Create Reusable AnimatedGlassCard Component

Build a component that:

- Uses the existing glass.tsx foundation for base glassmorphism
- Adds animated state transitions for active/inactive states
- Provides consistent styling tokens
- Handles press states with haptic feedback
- Is flexible enough to accommodate various card contents

### 2. Define Consistent Glassmorphism Tokens

Extract the proven values from LanguageSelectionScreen:

```javascript
// Active state (selected)
ACTIVE_BACKGROUND: "rgba(255,255,255,0.38)"
ACTIVE_BORDER_COLOR: "rgba(255,255,255,0.8)"
ACTIVE_BORDER_WIDTH: 1.5

// Inactive state
INACTIVE_BACKGROUND: "rgba(255,255,255,0.08)"
INACTIVE_BORDER_COLOR: "rgba(255,255,255,0.15)"
INACTIVE_BORDER_WIDTH: 1.5

// Animation (using reanimated)
TRANSITION_CONFIG: { damping: 20, stiffness: 150 }
```

### 4. Implementation Details for Each Screen Type

#### Standard Vertical Lists (Language, Goal, Mood, etc.)

- Replace Pressable/View wrappers with AnimatedGlassCard
- Maintain existing content structure (flags, text, icons, checkmarks)
- Preserve existing navigation and state update logic
- Add spring animations for state transitions

#### Horizontal Carousel (ThemeSelectionScreen)

- Wrap carousel items in AnimatedGlassCard
- Ensure horizontal paging and peek effects remain functional
- Apply glassmorphism to card containers
- Maintain existing orb and gradient designs inside cards

#### Complex Layouts (PaywallScreen, WelcomeScreen)

- Identify selectable elements (pricing cards, CTA buttons)
- Apply glassmorphism where appropriate
- Preserve existing complex layouts and functionality

### 5. Animation Specifications

Use react-native-reanimated for:

- Background color transitions: INACTIVE_BACKGROUND ↔ ACTIVE_BACKGROUND
- Border color transitions: INACTIVE_BORDER_COLOR ↔ ACTIVE_BORDER_COLOR
- Scale transitions: 0.98 ↔ 1.02 (subtle)
- Shadow intensity transitions (via glass layers if needed)
- Spring physics: damping 20, stiffness 150 for natural feel
- Press feedback: immediate scale to 0.95 on press, spring back

### 6. Glassmorphism Enhancement

Leverage existing glass.tsx by:

- Using GlassCard as base container for structural glass effect
- Overlaying animated state properties on top
- Ensuring the glass layers (wash, gradients, specular, shadows) remain intact
- Adding animated borders and background variations

### 7. Execution Sequence

1. Create AnimatedGlassCard component in src/lib/
2. Update LanguageSelectionScreen to use it (validation step)
3. Process remaining screens in logical groups:
   - Simple list screens (Language, Goal, Mood sets)
   - Complex screens (Paywall, Welcome, FreeTrial)
   - Specialized screens (ThemeSelection carousel)
4. Test each screen for functionality and visual consistency
5. Verify animation performance doesn't cause frame drops

### 8. Success Criteria

- All selectable cards share identical active/inactive visual language
- Smooth, spring-based transitions between states
- All existing functionality preserved (navigation, state updates, haptics)
- Unified premium glassmorphism feel across onboarding
- No degradation in animation performance
- LanguageSelectionScreen serves as validation that the approach works

## Key Improvements Over Current State

1. **Reusability** - Single component vs duplicated styling
2. **Animation** - Adds spring transitions where none existed
3. **Consistency** - Ensures identical tokens across all screens
4. **Maintainability** - Centralized glassmorphism definition
5. **Performance** - Optimized animated properties
